## 二、三大工作流详解

### 2.1 选股工作流 (Stock Selection)

```
输入: 候选股票池 (几十只)
          │
          ▼
┌─────────────────────────────────────────────┐
│         Phase 1: 数据准备                    │
│  - 获取K线数据                             │
│  - 判断市场状态(BULL/BEAR/NEUTRAL)         │
│  - 加载风险数据                            │
└──────────────────┬────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────┐
│         Phase 2: Agent并行分析               │
│  - TechnicalAgent: 均线/MACD/RSI/KDJ      │
│  - TacticAgent: 2560战法                   │
│  - RiskAgent: VaR/回撤/Beta                │
│  - PositionAgent: 凯利仓位/止损           │
│  - NewsAgent: 新闻情绪                     │
└──────────────────┬────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────┐
│         Phase 3: 委员会决策                  │
│  - 投票: BUY≥3 → 入选                     │
│  - 加权评分                               │
│  - 输出: 股票信号列表                      │
└──────────────────┬────────────────────────┘
                   │
                   ▼
输出: 符合条件的股票信号 (5-10只)
```

**Agent权重**:
- TechnicalAgent: 30%
- TacticAgent: 30%
- RiskAgent: 25%
- PositionAgent: 15%
- NewsAgent: 15%

---

### 2.2 买入工作流 (Buy Execution)

```
输入: 选股信号
          │
          ▼
┌─────────────────────────────────────────────┐
│         Phase 1: 仓位分配                     │
│  - 按评分排序                              │
│  - 分配仓位(置信度 × 30%, 最高25%)        │
│  - 最多5只股票                             │
└──────────────────┬────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────┐
│         Phase 2: 订单计算                    │
│  - 计算买入数量(整手)                      │
│  - 设置止损位(买入价 - 2×ATR)             │
│  - 设置目标位(买入价 + 3×ATR)              │
└──────────────────┬────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────┐
│         Phase 3: 订单执行                    │
│  - 记录订单                               │
│  - 创建持仓记录                           │
│  - 保存到positions.json                    │
└─────────────────────────────────────────────┘

输出: 持仓列表
```

---

### 2.3 卖出工作流 (Sell Execution)

```
输入: 持仓列表 + 实时价格
          │
          ▼
┌─────────────────────────────────────────────┐
│         Phase 1: 持仓检查                    │
│  - 更新当前价格                            │
│  - 计算浮动盈亏                           │
│  - 检测触发条件                            │
└──────────────────┬────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────┐
│         Phase 2: 卖出信号                    │
│  - 止损触发: 价格 ≤ 止损位                 │
│  - 目标达成: 价格 ≥ 目标位                 │
│  - 移动止盈: 盈利>10%且回撤5%             │
│  - 仓位再平衡: 亏损>-5%                   │
└──────────────────┬────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────┐
│         Phase 3: 卖出执行                    │
│  - 记录卖出历史                           │
│  - 计算实际盈亏                           │
│  - 更新持仓                               │
└─────────────────────────────────────────────┘

输出: 卖出记录 + 更新后的持仓
```

## 四、决策规则

### 4.1 选股决策

```
投票结果:
- BUY票 ≥ 3 → BUY (入选)
- AVOID票 ≥ 2 → AVOID (排除)
- HOLD票 ≥ 3 → HOLD (保留观察)
- 否则 → 按加权评分

加权评分 = Σ(Agent评分 × Agent权重)
```

### 4.2 卖出决策

```
触发条件:
1. 止损: 当前价 ≤ 止损位
2. 目标: 当前价 ≥ 目标位
3. 移动止盈: 盈利>10% AND 回撤≥5%
4. 仓位再平衡: 亏损>-5%
```
