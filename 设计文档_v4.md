## 一、系统概述

### 1.1 核心定位

虾米量化系统是一个**专业级A股智能决策系统**，通过多智能体协同、事件驱动工作流、完整量化因子模型，实现从选股到持仓管理的全流程自动化。**无任何功能孤岛，任何一个Agent的动作都会触发全网协调。**

## 三、多智能体协调框架 (Multi-Agent Coordination)

### 3.1 核心思想

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           事件驱动型Agent网络                              │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  任何一个Agent的动作都会触发:                                              │
│                                                                             │
│  1. 发布事件到Event Bus                                                   │
│  2. 所有订阅该事件的Agent同步收到                                           │
│  3. 每个Agent独立处理并可能发布新事件                                        │
│  4. 形成级联反应直到没有新事件                                              │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 3.2 Event Bus (事件总线)

| 功能 | 说明 |
|:---|:---|
| **订阅/发布** | Agent订阅事件，发布时所有订阅者同步收到 |
| **事件历史** | 记录所有事件，可追溯、可审计 |
| **防死锁** | RLock可重入锁，支持同一线程递归 |
| **防无限递归** | depth限制(最大5层) |

### 3.3 18个专业Agent

| Agent | 职责 | 订阅事件 |
|:---|:---|:---|
| **TechnicalAgent** | 技术分析(MA/MACD/RSI/KDJ) | price_update, new_stock |
| **TacticAgent** | 战术战法(2560战法) | price_update, technical_analysis |
| **RiskAgent** | 风险控制(VaR/止损) | position_opened, price_update |
| **NewsAgent** | 新闻事件分析 | news_received, breaking_news |
| **PositionAgent** | 仓位管理(凯利公式) | buy_decision, position_opened |
| **SectorAgent** | 板块轮动 | sector_impact, money_flow |
| **PolicyAgent** | 政策监控 | policy_announcement |
| **InternationalAgent** | 国际市场 | us_market, geopolitics |
| **MoneyFlowAgent** | 资金流向 | money_flow_update, north_flow |
| **SentimentAgent** | 市场情绪 | news_analysis |
| **FundamentalAgent** | 基本面分析 | financial_data |
| **VolatilityAgent** | 波动率分析 | price_update |
| **SeasonalAgent** | 季节效应 | time_based_events |
| **CorrelationAgent** | 相关性分析 | price_update |
| **BacktestAgent** | 回测验证 | buy_decision, sell_decision |
| **ExecutionAgent** | 交易执行 | final_decision |
| **MonitorAgent** | 实时监控 | all_events |
| **DecisionAgent** | 决策聚合 | 所有分析事件 |

### 3.4 事件列表

| 事件类型 | 触发者 | 订阅者 |
|:---|:---|:---|
| `new_stock` | 外部 | TechnicalAgent, SectorAgent |
| `breaking_news` | 外部 | NewsAgent, PolicyAgent |

| `buy_decision` | DecisionAgent | PositionAgent, ExecutionAgent |
| `sell_decision` | DecisionAgent | ExecutionAgent |
| `market_state_change` | 系统 | 所有Agent |

## 四、Agent协调示例

### 4.1 新闻事件协调流程

```
📢 breaking_news (央行降准利好)
    │
    ├─→ NewsAgent
    │       ├─ 分析情绪: sentiment=70
    │       ├─ 发布: news_analysis
    │       │       │
    │       │       ├─→ SectorAgent
    │       │       │       ├─ 更新板块: 银行+15, 券商+10
    │       │       │       └─ 发布: sector_change
    │       │       │
    │       │       ├─→ DecisionAgent
    │       │       │       └─ 收集意见后发布: final_decision
    │       │       │               │
    │       │       │               └─→ buy_decision
    │       │       │
    │       │       └─→ PositionAgent
    │       │               └─ 计算仓位: position_calculated
    │       │
    │       └─ 发布: sector_impact
    │               └─→ 所有关注板块的Agent收到通知
    │
    └─→ PolicyAgent
            └─ 分析政策影响并记录
```

### 4.2 持仓检查协调流程

```
📢 position_opened (贵州茅台持仓打开)
    │
    ├─→ RiskAgent
    │       ├─ 评估VaR: var_95=2800
    │       ├─ 发布: risk_assessment
    │       │       └─→ DecisionAgent 收集
    │       │
    │       └─ 监听价格更新检查止损
    │
    ├─→ PositionAgent
    │       └─ 更新总持仓价值
    │
    ├─→ DecisionAgent
    │       └─ 收集各Agent意见
    │
    └─→ MonitorAgent
            └─ 记录持仓状态
```

### 4.3 市场冲击协调流程

```
📢 us_market_update (美股大跌-2.5%)
    │
    ├─→ InternationalAgent
    │       ├─ 分析影响: A股-10
    │       └─ 发布: sector_impact(A股整体-10)
    │               │
    │               └─→ SectorAgent → DecisionAgent
    │
    └─→ geopolitics_update (地缘冲突)
            │
            ├─→ InternationalAgent
            │       └─ 发布: sector_impact(军工+15, 黄金+10)
            │
            └─→ NewsAgent
                    └─ 发布: news_analysis

所有影响汇总到DecisionAgent → 发布final_decision
```

## 五、37因子量化模型

### 5.1 因子体系

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          8大类 · 37个因子                                   │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  1️⃣ 价格因子 (7个): 动量、反转、波动率、Beta、Alpha、偏度、峰度         │
│  2️⃣ 资金流 (4个): 主力净流入、散户净流出、北向资金、融资融券              │
│  3️⃣ 情绪因子 (4个): 新闻情绪、社交情绪、分析师评级、机构调研              │
│  4️⃣ 政策因子 (4个): 货币政策、财政政策、行业政策、监管政策                 │
│  5️⃣ 基本面 (5个): P/E、P/B、ROE、营收增长、负债率                        │
│  6️⃣ 外部冲击 (5个): 美股、港股、汇率、大宗商品、地缘政治                  │
│  7️⃣ 市场结构 (4个): 板块轮动、资金配置、期现溢价、VIX指数                 │
│  8️⃣ 时间周期 (4个): 季节效应、周内效应、节假日、财报季                    │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 5.2 数学工具

| 类别 | 方法 | 用途 |
|:---|:---|:---|
| 统计学 | 均值/方差/偏度/峰度/VaR/CVaR | 风险评估 |
| 概率论 | 贝叶斯估计/蒙特卡洛/HMM | 不确定性量化 |
| 高等数学 | 梯度下降/牛顿法/傅里叶变换 | 优化/周期分析 |

## 六、自适应多轮权重优化

### 6.1 优化流程

```
第1轮: 最大似然估计 (MLE)
    ↓
第2轮: 贝叶斯更新 (融入先验)
    ↓
第3轮: L2正则化 (防止过拟合)
    ↓
第4轮: 梯度下降优化 (最大化夏普比率)
    ↓
第5轮: 市场状态调整 (牛/熊/中性)
    ↓
收敛检测 → 输出最优权重
```
